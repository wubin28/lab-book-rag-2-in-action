## Python中的两种导入方式

### 1. `import getpass` （第1行）
这种方式导入**整个模块**，使用时需要加上模块名作为前缀。

例如在第10行代码中：
```python
deepseek_api_key = getpass.getpass("")
```
这里调用的是 `getpass` 模块中的 `getpass()` 函数，需要写成 `getpass.getpass()`。

### 2. `from ... import ...` （第2-6行）
这种方式从模块中**只导入特定的类或函数**，使用时可以直接使用名称，不需要模块前缀。

例如：
- `from langchain_community.vectorstores import FAISS` - 只导入 `FAISS` 类
- 使用时直接写 `FAISS.from_texts(...)` ，而不是 `langchain_community.vectorstores.FAISS.from_texts(...)`

## 为什么第1行选择 `import getpass`？

这是**编码风格的选择**，两种写法都正确：

**选项1（当前代码）：**
```python
import getpass
deepseek_api_key = getpass.getpass("")
```

**选项2（也可以这样写）：**
```python
from getpass import getpass
deepseek_api_key = getpass("")
```

第一种写法的优点：
- 更清晰地表明 `getpass()` 函数来自 `getpass` 模块
- 避免命名冲突（如果代码中有其他名为 `getpass` 的变量或函数）

两种方式都是Python的标准用法，选择哪种主要取决于个人或团队的编码规范。
